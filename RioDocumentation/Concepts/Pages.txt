# Multiple Pages, Routing, and Guards

This guide will walk you through the process of setting up multiple pages,
handling routing, and using guards for access control in your Rio application.
By the end of this guide, you'll have a solid understanding of how to structure
your application for optimal navigation and security.

## Creating Pages

Creating a page is quite similar to creating a component. You have to do 3
things:

1. Add a new Python file to your project's `pages` folder.
2. Define a component in that file.
3. Decorate your component with the `@rio.page()` decorator.

Pages are pretty much regular components, except that pages go in the `pages`
folder instead of the `components` folder and that they're decorated with
`@rio.page()`. Here's an example:

```python
@rio.page(
    name="About Us",
    url_segment="about",
)
class AboutUsPage(rio.Component):
    def build(self) -> rio.Component:
        return rio.Markdown("This page provides information about us.")
```

Rio also has a deprecated class `rio.Page` (note the uppercase "P")
    that is easy to confuse with the `@rio.page` decorator. Make sure you're
    using the lowercase decorator instead of the uppercase class. The class is
    no longer used and will be removed in a future version of Rio.

The `url_segment` is set to `"about"`, making the page accessible at
`http://your.website/about`. Visiting that URL will display "This page provides
information about us.". For more information about this decorator and its
parameters, check out [the `@rio.page`
documentation](/docs/api/page).

Your file structure should now look similar to this:

```plain
my_project
├── __init__.py
├── components
|   ├── __init__.py
|   └── my_component.py
└── pages
    └── about_us.py  # <-- Here!
```

The file name doesn't matter, but the location of the file does!

### Example: Creating a Home Page

We've seen how to create a page that's accessible under the URL `/about`, but
what if we want to create a home page at the root URL (`/`)? This is done by
setting the page's `url_segment` to an empty string:

```python
@rio.page(
    name="Home",
    url_segment="",
)
class HomePage(rio.Component):
    def build(self) -> rio.Component:
        return rio.Markdown("Welcome to your Home Page!")
```

Now users can navigate between the home page at `/` and the about page at
`/about`. Each page is accessible through its unique URL segment, making routing
simple and intuitive.

### Sharing Content Between Pages

Oftentimes your website will have some elements that are identical on all pages,
like a navigation bar. Adding the navigation bar to every single page would be
tedious and error-prone.

We can solve this problem by using a custom `build` function for our app. The
output of the app's `build` function is visible on every page, no matter what.
We will instruct Rio to display the navigation bar at the top, and a
[`PageView`](/docs/api/pageview) (which will display the content
of the current page) below.

```python
from . import components as comps

def build() -> rio.Component:
    return rio.Column(
        comps.NavigationBar(),
        # The PageView displays the content of the current page
        rio.PageView(grow_y=True),
    )

app = rio.App(build=build)
```

## Nested Pages

Creating a nested page structure is essential for organizing your application
into sections and subsections. To create sub-pages, put them in a folder with
the same name as the file containing the parent page. For example, let's say we
want to create pages for the following URLs:

```plain
/
/docs
/docs/tutorial
/docs/api
```

Our file hierarchy would look like this:

```plain
pages
├── home_page.py           # URL: /
├── docs_page.py
└── docs_page              # (Folder)
    ├── docs_root_page.py  # URL: /docs
    ├── tutorial_page.py   # URL: /docs/tutorial
    └── api_docs_page.py   # URL: /docs/api
```

You may wonder what the point of the `docs_page.py` file is. This page is active
on every URL that starts with `/docs`. For example, if you're on
`/docs/tutorial`, then both the `docs_page` **and** the `tutorial_page` are
active. If you're on `/docs`, then the `docs_page` and the `docs_root_page` are
active. That means we can use the `docs_page` to place components that we want
to have on _all_ docs pages, like a side bar:

```python
@rio.page(url_segment="docs")
class DocsPage(rio.Component):
    def build(self) -> rio.Component:
        return rio.Row(
            comps.SideBar(),
            # The PageView is responsible for displaying
            # the currently active sub-page
            rio.PageView(grow_x=True),
        )
```

Make sure to use a `PageView` on your parent page, otherwise the child pages
won't be displayed! But what does the `PageView` do on the URL `/docs`? It needs
a child page to display, otherwise we'll get an error. That's where the
`docs_root_page` comes in. Defining this page with `url_segment=""` creates a
child page that is only active on the URL `/docs`:

```python
@rio.page(url_segment="")
class DocsRootPage(rio.Component):
    def build(self) -> rio.Component:
        return rio.Markdown("You're currently on `/docs`!")
```

## Navigating / Routing

Navigation is a crucial part of any web application. In Rio, there are several
ways to navigate between pages: programmatically, using the
[`rio.Link`](/docs/api/link) component, or via direct URL access.

### Navigating to a Page Programmatically

You can navigate to a page programmatically using the `navigate_to` method
available in the `Session` object. This is useful when you need to trigger
navigation as a result of some user interaction, like clicking a button.

```python
class MyComponent(rio.Component):
    def navigate_to_home(self):
        # Perform any pre-navigation logic here
        self.session.navigate_to("/") # Navigate to the root page

    def build(self) -> rio.Component:
        return rio.Button(
            "Go to Home",
            on_click=self.navigate_to_home,
        )
```

In this example, clicking the button triggers the `navigate_to_home` method,
which directs the user to the Home page.

### Navigating with the `rio.Link` Component

For simpler cases, you can use the [`rio.Link`](/docs/api/link)
component to create a link to a page:

```python
rio.Link("Home", "/")
```

### Combining Links and Buttons

You can also use the `rio.Link` component to wrap other components, such as
buttons, to create interactive navigation elements:

```python
rio.Link(
    rio.Button("Home"),
    target_url="/",
)
```

This approach combines the visual appearance of a button with the functionality
of a link.

### Navigating Directly via URL

Users can navigate directly to a specific page by entering the corresponding URL
in their browser. For example, to visit the About Us page we created earlier,
they would go to `http://your.website/about`.

## Using Guards

Guards are an essential feature for controlling access to certain pages in your
application. They allow you to implement logic that checks whether a user has
the necessary permissions to access a page.

### How Guards Work

A guard is a function that takes a `GuardEvent` object as an argument. Based on
the logic within the guard, it returns a string (the `url_segment`) to redirect
unauthorized users or `None` to grant access.

In more details see our
[Authentication](/examples/authentication) example and the [API
docs for GuardEvent](/docs/api/guardevent).

### Example: Creating a Guard

This example demonstrates a guard that checks if the user is logged in. If the
user is not logged in, they are redirected to the login page.

```python
def guard(event: rio.GuardEvent) -> str | None:
    """
    Create a guard that checks if the user is already logged in.

    ## Parameters

    `event`: The event that triggered the guard containing the `session`
        and `active_pages`.
    """
    if is_logged_in(event.session):
        # User is logged in, no redirection needed
        return None
    else:
        # User is not logged in, redirect to the login page
        return '/login'
```

### Applying a Guard to a Page

To protect a page with a guard, simply add the `guard` parameter to the
`@rio.page` decorator:

```python
@rio.page(
    name="App Page",
    url_segment="app",
    guard=guard, # Apply the guard function
)
class AppPage(rio.Component):
    def build(self) -> rio.Component:
        return rio.Text("This page is protected. Only authorized users can view this content.")
```

### Use Cases for Guards

- **Authentication:** Prevent unauthenticated users from accessing specific
    pages.
- **Authorization:** Ensure users have the correct roles or permissions before
    accessing certain features.
- **Custom Logic:** Implement any custom access logic, such as time-based
    access restrictions or feature flags.

## Creating Pages dynamically

Sometimes you might want to create pages dynamically, which is difficult to do
with the `@rio.page` decorator. Rio offers two solutions for this:

1. Use [pages with URL parameters](/docs/howto/route-parameters).
2. Pass a list of pages into the `App` constructor.

Here, we will focus on the 2nd option.

The `App` has a `pages` parameter, which allows you to pass in a list of pages.
There are two classes that can be used as input here:

- [`rio.ComponentPage`](/docs/api/componentpage) is a regular
    page, with a name, url segment, build function, a list of sub-pages, etc.
- [`rio.Redirect`](/docs/api/redirect) redirects the user to a
    different URL.

Here's an example with nested pages and a redirect:

```python
app = rio.App(
    build=lambda: rio.Column(
        rio.Text("Welcome to my app!"),
        rio.PageView(grow_y=True),
    ),
    pages=[
        rio.ComponentPage(
            name="Home",
            url_segment="",
            build=lambda: rio.Text("Welcome to your Home Page!"),
        ),
        rio.ComponentPage(
            name="Blog",
            url_segment="blog",
            build=rio.PageView,
            children=[
                rio.ComponentPage(
                    name="Welcome",
                    url_segment="",
                    build=lambda: rio.Text("Welcome to our blog!"),
                ),
                rio.Redirect(
                    url_segment="home",
                    target="/blog",
                )
            ]
        ),
    ],
)
```

This will display "Welcome to your Home Page!" when navigating to the root URL,
or "Welcome to our blog!" when navigating to `/blog`. Navigating to `/blog/home`
will redirect to `/blog`.
