# Attribute Bindings

Graphical user interfaces need data flow in 2 directions: From the program to
the user (i.e. output), and from the user to the program (i.e. input). Handling
output is easy enough, but what about input?

```python
class MyComponent(rio.Component):
    user_name: str = 'John'

    def build(self) -> rio.Component:
        return rio.Column(
            # Output is easy
            rio.Text(f'Your name is: {self.user_name}'),
            # But how do we get input?
            rio.TextInput()
        )
```

Attribute bindings allow a child component to synchronize its state with the
parent component. For example, we can use it to sync the value of the
`TextInput` with our `user_name` attribute. To create an attribute binding,
simply pass `self.bind().attribute_name` to an input element like this:

```python
class MyComponent(rio.Component):
    user_name: str = 'John'

    def build(self) -> rio.Component:
        return rio.Column(
            # Output is easy
            rio.Text(f'Your name is: {self.user_name}'),
            # And with attribute bindings, input is easy too!
            rio.TextInput(self.bind().user_name)
        )
```

Now, whenever the value of the `TextInput` changes, the value of
`self.user_name` will automatically change as well.

## Bindings are instructions, not values

A common mistake is to treat attribute bindings - in particular, the output of
`self.bind().attribute_name` - as if they were values. For example, you might be
tempted to write this:

```python
rio.TextInput(f'Your name is: {self.bind().user_name}')
```

This doesn't work, because `self.bind().user_name` is an _instruction_ that
tells the `TextInput` to synchronize its value with our `user_name` attribute.
If you want to access the current _value_ of `user_name`, you must write
`self.user_name` instead. Remember: Use `self.user_name` for output, and
`self.bind().user_name` for input. You can't do both at the same time.

## Attribute Bindings in custom components

Attribute Bindings aren't reserved for builtin rio components - _all_ components
support attribute bindings. And the best part is that you don't have to do
anything at all! Check out this custom number input:

```python
import random

# There's nothing special about this component...
class CustomNumberInput(rio.Component):
    value: int

    def random_value(self) -> None:
        self.value = random.randrange(0, 100)

    def build(self) -> rio.Component:
        return rio.Row(
            rio.NumberInput(self.bind().value, decimals=0),
            rio.Button('Random', on_press=self.random_value),
        )

class MyComponent(rio.Component):
    number: int = 0

    def build(self) -> rio.Component:
        return rio.Column(
            # ...and yet it supports attribute bindings!
            CustomNumberInput(self.bind().number),
            rio.Text(f'The number is {self.number}'),
        )
```
