Text

Displays unformatted text.

`Text` displays text without any formatting, making it one of the most
commonly used components in Rio.

While the text itself is unformatted, you can still control the style of
the text using the `style` attribute. This allows you to change the font
size, color, and more.

text
`str`

The text to display.

selectable
`bool`

Whether the text can be selected by the user.

style
`Literal['heading1', 'heading2', 'heading3', 'text', 'dim'] | rio.TextStyle`

The style of the text. This can either be a [`TextStyle`](/docs/api/textstyle) instance,
or one of the built-in styles: `"heading1"`, `"heading2"`, `"heading3"`,
`"text"` or `"dim"`.

justify
`Literal['left', 'right', 'center', 'justify']`

Controls the alignment of the text within each individual line.

wrap
`bool | Literal['ellipsize']`

Deprecated in favor of [`overflow`](/docs/api/markdown).

overflow
`Literal['nowrap', 'wrap', 'ellipsize']`

What to do with long text. If this is `"nowrap"`, the text will
be as wide as it needs to be to fit without wrapping. If `"wrap"`, the
text will wrap to the next line when running out of horizontal space.
Finally, if `"ellipsize"`, the text will be truncated when there isn't
enough space and an ellipsis (`...`) will be added.

`font`: The [`Font`](/docs/api/font) to use for the text. When set to `None`, the default font
for the current context (heading or regular text, etc) will be used.

font
`rio.Font | None`

The name of the font to use. Overrides the `font` from the `style`.

fill
`rio.SolidFill | rio.LinearGradientFill | rio.ImageFill | rio.Color | rio.RadialGradientFill | None`

The fill (color, gradient, etc.) for the text. Overrides the `fill`
from the `style`.

font_size
`float | None`

The font size. Overrides the [`font_size`](/docs/api/textstyle) from the `style`.

italic
`bool | None`

Whether the text is *italic* or not. Overrides the `italic` from
the `style`.

font_weight
`Literal['normal', 'bold'] | None`

Whether the text is normal or **bold**. Overrides the
[`font_weight`](/docs/api/textstyle) from the `style`.

underlined
`bool | None`

Whether the text is underlined or not. Overrides the
[`underlined`](/docs/api/textstyle) from the `style`.

strikethrough
`bool | None`

Whether the text should have a line through it. Overrides
the [`strikethrough`](/docs/api/textstyle) from the `style`.

all_caps
`bool | None`

Whether the text is transformed to ALL CAPS or not. Overrides
the [`all_caps`](/docs/api/textstyle) from the `style`.

key
`str | int | None`

A unique identifier for this component. If two components with the
same key are present during reconciliation they will be considered
the same component and their state will be preserved. If no key is
specified, reconciliation falls back to a less precise method, by
comparing the location of the component in the component tree.

min_width

request during layouting. The component will never be smaller than this

Please note that the space a [`Component`](/docs/api/component) receives during layouting may

## Examples

Here's a minimal example for displaying text. Just pass it a string:

```python
rio.Text("Hello, world!")
```

To change the style of the text, you can pass a [`TextStyle`](/docs/api/textstyle) instance:

```python
rio.Text(
    "Hello, world!",
    style=rio.TextStyle(
        font_size=3,
        # Text Style has a lot of optional parameters. Have a look at its
        # docs for all details!
        #
        # https://rio.dev/docs/api/textstyle
    ),
)
```
